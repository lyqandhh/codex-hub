# Codex HUD 设计规格

## 目标

构建一款原生 macOS 悬浮组件，让用户无需打开 Codex 设置或点击菜单栏，即可持续看到 Codex 的每周剩余额度、重置日期与可用重置次数。

成功标准：组件长期置顶但不抢注意力；额度数据自动更新；读取失败时不展示误导性数据；整个应用无需 OpenAI API Key，也不上传本机账号或使用数据。

## 产品形态

- 应用名：Codex HUD。
- 项目位置：`/Users/mantou/Projects/CodexHUD`。
- 平台：macOS，优先适配当前 Apple Silicon 设备。
- 技术：SwiftUI 负责界面，AppKit 负责无边框悬浮窗口、窗口层级、拖动和鼠标穿透。
- 应用不显示 Dock 图标，主要界面是屏幕右上角的单个悬浮胶囊。

## 视觉设计

采用已选定的“极简横向胶囊”方向：

- 默认尺寸约 390 × 52 pt，允许内容需要时小幅扩展。
- 深色半透明毛玻璃背景，圆角胶囊轮廓，使用系统字体。
- 主信息依次为 `1周 99%`、分隔点、`重置 ×3`、分隔点、`7月20日`。
- `1周`表示当前周额度窗口，百分比始终表示剩余额度。
- `重置 ×N`始终显示，包括 `重置 ×0`，避免用户将隐藏状态误解为读取失败。
- 日期显示当前周额度窗口的重置日期；不足 24 小时时改为相对时间，例如 `18小时后`。
- 右侧状态点表示数据新鲜度：绿色为实时、黄色为使用缓存、灰色为尚无有效数据。
- 底部 3 pt 进度线表示当前周剩余额度。
- 进度线颜色按剩余额度变化：充足时绿色，中等时黄色，低于 20% 时橙红色。
- 不使用持续动画；刷新成功只允许非常轻微的状态点过渡。

## 交互

- 窗口始终置于普通应用窗口之上，默认出现在当前主屏幕右上角安全区域。
- 用户可拖动胶囊，应用持久化最后位置。
- 双击胶囊切换鼠标穿透；切换后给出短暂、非模态的视觉反馈。
- 鼠标穿透开启时，提供全局快捷键恢复交互，避免组件无法操作。
- 右键菜单包含：立即刷新、鼠标穿透、透明度、登录时启动、恢复默认位置、退出。
- 不弹出常规低额度通知；低于 20% 时只改变进度线颜色。

## 数据读取

数据层通过独立的 `QuotaProvider` 协议与界面解耦。

首选实现调用本机 Codex `app-server` 的 `account/rateLimits/read` 能力，读取当前周窗口的剩余比例、重置时间，以及 `rateLimitResetCredits.availableCount`。调用在本机执行，不使用 API Key，不向第三方发送数据。

首版只读取重置次数，绝不调用 `account/rateLimitResetCredit/consume`，避免误触发不可逆的额度重置操作。

应用启动时立即刷新，此后默认每 60 秒刷新。手动刷新不改变自动刷新节奏。

若实时读取失败：

1. 保留并展示最后一次成功数据。
2. 状态点切换为黄色，并记录数据更新时间。
3. 连续失败且没有任何历史数据时，显示 `额度暂不可用`，不得用 0% 或 100% 代替未知值。
4. 下次定时刷新继续重试，不频繁拉起新进程。

由于 Codex 本地协议可能随版本变化，额度调用和响应解析必须集中在单独模块，并使用样例响应测试覆盖。

## 组件边界

- `CodexHUDApp`：应用生命周期与设置入口。
- `FloatingPanelController`：窗口层级、位置、拖动、穿透和多显示器处理。
- `QuotaProvider`：额度数据来源协议。
- `CodexAppServerQuotaProvider`：Codex 本地实时额度实现。
- `QuotaStore`：刷新调度、缓存、错误状态和可观察界面状态。
- `QuotaCapsuleView`：胶囊界面，仅消费格式化后的视图状态。
- `PreferencesStore`：透明度、穿透、刷新频率、窗口位置和登录启动设置。

## 状态与格式化

内部保存周额度“剩余比例”，范围限制为 0...1；界面显示为整数百分比。若协议返回的是已用比例，数据适配层负责转换，视图层始终只接收剩余比例。

重置次数使用非负整数；字段不存在与数值为 0 必须区分。字段不存在时显示 `重置 --` 并使用黄色状态点，真实为 0 时显示 `重置 ×0`。

重置时间使用用户当前时区格式化：24 小时内显示相对时间，超过 24 小时显示月日；跨年时追加年份。

## 隐私与安全

- 不读取或展示认证令牌。
- 不打印 Codex 本地服务的完整原始响应。
- 不引入分析、遥测或远程日志。
- 不开放网络监听端口。
- 缓存只保存周额度数值、重置时间、可用重置次数和更新时间。

## 验证

- 单元测试覆盖额度响应解析、已用到剩余的转换、百分比边界、重置次数缺失与为 0 的区别、相对时间格式和失败回退。
- 使用可注入的假 `QuotaProvider` 验证刷新与缓存状态。
- 构建后运行原生应用，检查无 Dock 图标、窗口置顶、拖动、穿透恢复、多显示器位置和右键菜单。
- 真实连接本机 Codex 后，对照 Codex 自身额度页面核对周额度百分比、重置日期和可用重置次数。
- 视觉检查包括浅色与深色桌面背景、不同缩放比例以及长中文重置文本。

## 首版范围外

- Windows 与 Linux 版本。
- 多 Codex 账号切换。
- 历史消耗图表和成本统计。
- 云同步、团队共享与远程访问。
- 自定义主题市场或复杂布局编辑器。
